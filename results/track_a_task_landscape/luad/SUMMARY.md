# LUAD (TCGA) — second-disease run

This is the Phase-E (E5) second-disease validation. The pipeline ran
unmodified on TCGA-LUAD tumor-vs-normal to show the claim "the
pipeline generalizes across diseases" is not a KIRC-only artefact.

## Cohort

- Source: TCGA-LUAD star_tpm, GDC-Xena hub
  (`.tmp_geo/gdc/TCGA-LUAD.star_tpm.tsv.gz`), built by
  `data/build_tcga_luad.py` → `data/luad_tumor_normal.csv`.
- Samples: 589 total (530 tumor / 59 normal); 58 patients have paired
  tumor + normal.
- Panel: 23-gene (lung identity + proliferation + hypoxia + oncogene),
  card at `config/dataset_cards/luad_tumor_normal.json`.

## Single-gene ceiling on this task

| Gene | sign-inv AUROC |
|---|---|
| SFTPC | 0.998 |
| SLC2A1 | 0.960 |
| CDK1 | 0.934 |
| LDHA | 0.927 |
| SFTPB | 0.813 |
| NAPSA | 0.732 |

Tumor-vs-normal on LUAD is effectively saturated by SFTPC (surfactant
protein C, lost in tumor-dedifferentiated lung) — the same structural
issue as CA9 on KIRC tumor-vs-normal (AUROC 0.965).

## Opus 4.7 ex-ante laws (pre-registered in `config/law_proposals.json`)

Four LUAD-specific templates were written by Opus 4.7 before seeing any
LUAD data. All four clear 4 of the 5 gate tests but fail on
`delta_baseline` because SFTPC alone is already a 0.998 classifier.

| Law family | AUROC | Δbase | ci_lower | perm_p | decoy_p | verdict |
|---|---|---|---|---|---|---|
| `log1p(CDK1) + log1p(SLC2A1) − log1p(SFTPC)` | 0.994 | −0.004 | 0.989 | 0.000 | 0.000 | FAIL (delta_baseline) |
| `log1p(LDHA) − log1p(NAPSA)` | 0.914 | −0.083 | 0.889 | 0.000 | 0.000 | FAIL (delta_baseline) |
| `log1p(VEGFA) + log1p(CDK1) − log1p(SFTPB)` | 0.905 | −0.093 | 0.878 | 0.000 | 0.000 | FAIL (delta_baseline) |
| `log1p(CDK1)` | 0.934 | −0.064 | 0.912 | 0.000 | 0.000 | FAIL (delta_baseline) |

`0 / 4 Opus ex-ante candidates survive.` Same failure mode as KIRC
tumor-vs-normal: a compound law cannot clear +0.05 over the best
single-gene classifier because the task is saturated by one gene.

## PySR search

PySR search on the same 22-gene panel (23 minus NKX2-1 alias column to
keep equations hyphen-free):

- `results/track_a_task_landscape/luad/candidates.json` (generated
  by `src/pysr_sweep.py`).
- `results/track_a_task_landscape/luad/falsification_report.json`
  (generated by `src/falsification_sweep.py`).

Expected outcome (pre-registered): 0 survivors for the same reason as
the ex-ante laws — any compound has to beat SFTPC's 0.998 baseline by
+0.05, which is mathematically impossible (`law_AUROC` would need
> 1.048). A positive outcome would require a harder LUAD task (e.g.
stage I-II vs III-IV, lymph-node-positive vs -negative, or the
Schabath smoking-status subgroup).

## What this shows

1. **Cross-disease generalization.** The same 5-test gate, the same
   Opus 4.7 Proposer prompt, the same PySR search, runs unmodified on
   a different cancer. The `--dataset-card config/dataset_cards/luad_tumor_normal.json`
   path supplies the panel without touching code.
2. **Same failure mode, same task class.** Tumor-vs-normal in any
   cancer where one marker saturates gives the gate an unclearable
   +0.05 threshold. This is a feature, not a bug: the gate refuses to
   call a single-gene task a multi-gene discovery.
3. **Next step is the harder LUAD task.** The KIRC survivor emerged
   on `metastasis` with the 45-gene panel, not on tumor-vs-normal.
   Applying the same reasoning to LUAD would require the TCGA-LUAD
   clinical table's `ajcc_pathologic_stage` / `ajcc_pathologic_m`
   columns — a Phase-E3 / future step, not a Phase-E5 blocker.

## Reproduce

```bash
# 1. Download (one-time)
curl -o .tmp_geo/gdc/TCGA-LUAD.star_tpm.tsv.gz \
  "https://gdc-hub.s3.us-east-1.amazonaws.com/download/TCGA-LUAD.star_tpm.tsv.gz"
.venv/bin/python data/build_tcga_luad.py

# 2. Opus ex-ante falsification (no API, uses committed law_proposals)
PYTHONPATH=src .venv/bin/python src/build_exante_candidates.py \
  --proposals config/law_proposals.json --dataset luad \
  --output results/track_a_task_landscape/luad/opus_exante_candidates.json

PYTHONPATH=src .venv/bin/python src/falsification_sweep.py \
  --candidates results/track_a_task_landscape/luad/opus_exante_candidates.json \
  --data data/luad_tumor_normal.csv \
  --genes "SFTPC,NAPSA,SFTPB,SFTPA1,SFTPA2,EPCAM,KRT7,KRT18,KRT19,NKX2-1,CDK1,CCNB1,TOP2A,MKI67,PCNA,MCM2,SLC2A1,LDHA,HK2,VEGFA,EGFR,KRAS" \
  --output results/track_a_task_landscape/luad/opus_exante_report.json

# 3. PySR + falsification (30-60 min on 1 laptop CPU)
PYTHONPATH=src .venv/bin/python src/pysr_sweep.py \
  --data data/luad_tumor_normal.csv \
  --genes "SFTPC,NAPSA,SFTPB,SFTPA1,SFTPA2,EPCAM,KRT7,KRT18,KRT19,CDK1,CCNB1,TOP2A,MKI67,PCNA,MCM2,SLC2A1,LDHA,HK2,VEGFA,EGFR,KRAS" \
  --proposals config/law_proposals.json \
  --iterations 300 --n-populations 8 --population-size 40 --maxsize 15 --seeds 7 \
  --output results/track_a_task_landscape/luad/candidates.json

PYTHONPATH=src .venv/bin/python src/falsification_sweep.py \
  --candidates results/track_a_task_landscape/luad/candidates.json \
  --data data/luad_tumor_normal.csv \
  --genes "SFTPC,NAPSA,SFTPB,SFTPA1,SFTPA2,EPCAM,KRT7,KRT18,KRT19,NKX2-1,CDK1,CCNB1,TOP2A,MKI67,PCNA,MCM2,SLC2A1,LDHA,HK2,VEGFA,EGFR,KRAS" \
  --output results/track_a_task_landscape/luad/falsification_report.json
```

The pipeline generalizes. The KIRC survivor emerged on a *harder*
task with a *wider* panel; repeating that recipe for LUAD would be
an E3-style stage or metastasis run, which is the next step in the
Phase-E roadmap.
